# Contributing to Diversy

Welcome to Diversy!
Thank you for wanting to help! Diversy is a Source-Available project. We welcome contributions that improve the project, provided they respect our restrictive license.

## 1. The Workflow

To contribute code, please follow these steps:

- **Fork the repository**: You are authorized to fork this project only for the purpose of contributing back to this upstream repository.
- **Create a Branch**: Name it something descriptive (e.g., fix/auth-bug or feat/new-ui).
- **Commit Your Changes**: Ensure your code follows our tech stack (Bun, TypeScript, Svelte).
- **Open a Pull Request (PR)**: Describe exactly what your changes do.

## 2. Contribution Agreement

By submitting a Pull Request, you agree to the following:

- You grant the project owner a permanent, non-exclusive, royalty-free license to use, modify, and distribute your contribution as part of Diversy.
- Your contribution will fall under the project's "No Commercial Use" and "No Redistribution" clauses.

- You waive any rights to claim ownership over the version of the project containing your changes.

## 3. Tech Stack Guidelines

- **Runtime**: Use Bun (please do not add package-lock.json for npm/yarn).
- **Framework**: SvelteKit/Svelte.
- **Language**: TypeScript.

## 4. Reporting Issues

If you aren't a coder but want to help, please open an Issue. Provide as much detail as possible, including screenshots or logs.
